# OllyDbg

# 经验

  • 在一个函数上下断点的时候，函数可能多次被不同的地方调用，我们关心的一次调用，要去观察，比如 createMutex 函数，很多线程的共享的地方也要调用这个函数，我们关心的调用是参数里有 wechat_mutex 这个字符串的时候这个函数调用，然后跳回到它调用者，再去观察。

  • 在找关键的跳转语句时候，要找出跳转的不同情况去观察。 比如微信多开， 那么可以先开一个微信，在这个跳转处看一下第一次打开是否跳转，然后再打开第二次的时候再去看这个地方是否跳转。如果不跳转就说明找大了关键的跳转语句。

  • 在找关键的跳转语句的时候 可以跳出这个函数到它的调用者中去看。

# 命令

  • ctrl g 找到函数名，下断点

  • f9 运行直到断点 Ctrl + f9 运行到当前函数结束准备返回前

  • f8 单步运行

  • F7单步进入

  • f4 运行到当前位置

  • + - * 往回退，前进，回到当前位置 可以查看函数是从哪里调用来的

  • 回车 跟随进入，在函数返回前可以进入函数调用后的ip

  • 空格 修改当前位置的指令代码

# 数据查看

  • (dd ebp) dump doubleword 先查看32位数， 双击在这行，这行是ebp， 把这行标示成紫色，已经把数据根据高低字节拼出来， 0039f954

  • (dw ebp) dump word 如果用16位数查看， 这个地址显示位 f954 0039 , 左边0039 是高位放在高地址，左边是高。

  • (db ebp) dump byte 如果用8位数查看， 这个地址显示位 54 f9 39 00, 左边00 是高位放在高地址，左边是高。

  • dc dump character 用ascii码显示
