# 编译器优化

  • 用 push ecx, pop ecx （指令都只有一个字节） 来替换 add/sub esp,04 (3个字节) 节约空间

  • 把常用的函数比如printf的地址 压入寄存器， mov esi,dword ptr ds:[<&MSVCR100.printf>> 提高速度

  • 尽量把变量放在寄存器，少用栈。

  • 函数return后，有时候栈不平衡，直接给变量使用。

# cdel stdcall fastcall 约定

  • __cdel 参数从右到左入栈，由调用者进行栈平衡，add esp 8, c 与 c++ 默认都使用这个约定。 可变参数的函数必须用 cdel

  • __stdcall 参数从右到左入栈，由被调用者返回的时候进行栈平衡 ret 8. 一般系统的api使用这个。

  • __fastcall 前两个参数由寄存器传递，第3个参数用入栈的方式，也是由被调用者进行栈平衡 如果参数超过2个的时候。主要用与频繁调用的函数而且参数<=2个的时候使用.

# and和or的使用

  • and的用法

    • 当要给某个位设置成0 的时候 就给那个位用0和

  • or的用法

    • 当要给某个位设置成1的时候就给那个位 用1 or

# 数字要注意的问题

  • 当数字首个是字母的时候要加上0。 如 mov 0b800H 可以，不加0就编译不过

  • 数字是十进制就不用加h，十六进制就要加h

# 为什么要编译成obj

把一个大的源代码分成多个源文件，这样修改一些内容的时候，不用全部从头编译，只编译修改的文件为新的obj文件然后link。
