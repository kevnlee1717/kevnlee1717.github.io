# docker

# 框架

  • 先用 Dockerfile 文件定义一个image， 然后从这个image build一个contianer来给Docker引擎去运行

    • run 是在building的时候临时的container里运行的命令
    • cmd 是在image运行为一个正式的conatiner后，在container里运行的命令

  • 可以用dockerfile做两个系统，一个是大的功能齐全把所有复杂工作都做掉，并生成一个结果在一个目录下。比如大系统用的是Ubuntu， 然后另外一个系统可以是alpine，直接复制这个结果来自大系统。这样平常用小系统就可以看结果。 copy --from=id 这里的id就是大系统的id

  • 即使是同一个image run出来不同的container，里面的文件等都不相通, 每个container都是独立的.

  • 可以把container想象成为一个可以执行文件，在操作系统里运行的实例，一个可执行文件可以有多个实例，每个实例互不相通。

  • container运行后，不管是运行状态还是停止状态，这个container都是存在的，只有remove后，才是销毁，在没有销毁的时候可以随时开始，暂停。里面新增加的内容也都存在。

  • 当container运行起来后，里面如果安装了软件有了新的内容，可以用 commit命令 生产一个新的image。

  • run的时候 -d可以把container运行到后台，然后可以 attach 命令进入这个container。

  • run 的时候也可以用 -rm 运行好相应的命令后就把container停止。

  • container之间的通信可以用 port 也可以用link 当link的时候 主机的ip是写在/etc/hosts文件里的，所以如果两个机器同时启动同时停止，用link的方式可以，不用通过宿主机器中转。

  • 如果要用link而且在主机remove后，再用同样名字建立container的时候，其他container还能连接，要主机和其他容器都加入同一个 private network，用 docker crate network network-name 建立， 主机和其他contianer在run 的时候都用 --net netowrk-name来启动。

  • 主机和container， container和container之间的文件共享

    • 一种是container之间共享数据，当没有container使用的时候，这些数据就丢失了。 第一个container建立时候 run -v contaier-folder ，其他container要使用 run --volumes-from container-name，这里的container-name只要是能访问这个目录的container的名字就可以。

    • 一种是绑定模式， 主机的文件系统主机和container之间的共享，可以在run的时候 加上 -v host-folder:contaienr-folder 来启动，这样host-folder就是共享文件夹。不会丢失数据。但是对操作系统有依赖。

    • 最后是用 命名的 volume， 由docker管理，在container运行的时候，映射到目录， 可以用 docker volume ls 来查看，还可以管理，移动到不同的操作系统和平台。

  • docker-compose 把定义和运行都定义在一个文件里，自动运行。

    WARNING

    用docker-compose up -d 运行起来的container， 在后续的 docker 命令 要用到container id 的时候，是第一行定义的id

    
app: # 这个才是运行起来后的ID
    build:
    context: .
    dockerfile: Dockerfile
    image: digitalocean.com/php
    container_name: app2   # 这个只是ps 里看到的 
    restart: unless-stopped
    tty: true
    environment:
    SERVICE_NAME: app2  # 这个也是
    SERVICE_TAGS: dev

# Dockerfile

DockerFile 是用来建立container

其中的copy命令的第二个参数是container里的WORKDIR


# 从一个官方的node镜像上建立
FROM node

# 设置container里的工作目录
WORKDIR /usr/src/app

# 从Dockerfile所在的本机当前目录，
# 复制到container刚刚建立的目录里
COPY pakage*.json ./

# 在container里运行 npm install 安装所有的node依赖
RUN npm install

# 复制当前目录下所有的文件到
# container里刚刚建立的工作目录
# 复制的时候根据 .ignore文件,跳过文件和目录
COPY . . 

# 在container里开放4000端口
# 在container里运行 npm start
EXPOSE 4000
CMD  ["npm", "start"]

# docker-compose.yml

compose文件可以对多个contianer进行定义, 对依赖进行定义， 对image运行的参数时候定义，所以一个文件就进行了image的定义，又对运行时候进行了定义，等于对 docker build 和 docker run 进行了包装，同时还对依赖进行了定义，根据依赖关系，所需要的container先启动

  • 定义依赖关系 links
  • build image时候需要暴露的端口 expose
  • 运行时候端口的映射 ports
  • 运行时候需要的共享文件夹 volumes
# app是跟 dockerfile里的一样？
app:
   
   #容器名，跟 dockerfile里面的一样？
   container_name: app

   # 有错误的时候总是重启
   restart: always

   # 从当前目录获取dockerfile
   build .

   # 运行的时候需要打开的端口
   ports:
    - "4000:4000"
   
   # 依赖于哪个容器，需在后面有这个定义
   links:
       - mongo

#mongo定义
mongo:
   
   # 容器名,运行起来后的名字？
   container_name: mongo

   # 镜像名，获取官方的mongo镜像
   image: mongo

   # 创建端口,相当于给mongo的dockerfile里配置端口
   expose: 
    - "27017"
   
   # 建立一个共享，格式？
   volumes:  
    - ./data:/data/db

   # 运行起来后的端口映射
   ports: 
    - "27017:27017"

version: 3
services:
    # app是跟 dockerfile里的一样？
    app:
        
        #容器名，跟 dockerfile里面的一样？
        container_name: api

        # 有错误的时候总是重启
        restart: always

        # 从api目录获取dockerfile
        build: ./api

        # 运行的时候需要打开的端口
        ports:
        - "4000:4000"
        
        # 依赖于哪个容器，需在后面有这个定义
        depends:
            - mongo

    client:

        #容器名，跟 dockerfile里面的一样？
        container_name: client

        # 有错误的时候总是重启
        restart: always

        # 从client目录获取dockerfile
        build: ./client

        # 运行的时候需要打开的端口
        ports:
        - "4000:4000"
        
        # 连接于哪个容器,需要有定义
        links:
            - app

    #mongo定义
    mongo:
        
        # 容器名,运行起来后的名字？
        container_name: mongo

        # 镜像名，获取官方的mongo镜像
        image: mongo

        # 创建端口,相当于给mongo的dockerfile里配置端口
        expose: 
        - "27017"
        
        # 建立一个共享，格式？
        volumes:  
        - ./data:/data/db

        # 运行起来后的端口映射
        ports: 
        - "27017:27017"
