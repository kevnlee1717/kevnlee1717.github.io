# 属性绑定

每个组件里的属性都是可以跟 data 里的变量去绑定的，这才是vue强大和灵活的地方

<scroll-view scroll-x class="scroll-row border-bottom border-white" :scroll-into-view="scrollIntoView" scroll-with-animation>
	<view v-for="(item,index) in subpages" :key="index" class="scroll-row-item px-5 py-3"
		:class="currentPage === index ? 'comm-color font-weight-bold font-lg' : ''"
		@click="changePage(index)"
		:id="'tab'+index"
		>{{item.name}}</view>
</scroll-view>

# 总结

  • 样式的导入是在app.vue，但是对这些样式的覆盖在page.json文件里，全局的样式覆盖在 globalstyle, pages里文件自己的定义又可以覆盖，，到具体的页面文件里的 style又可以进行覆盖

# 技巧

  • 路径快速输入：在需要路径的地方，输入关键字，可以快速显示路径，比如在需要输入页面路径，或者图标路径的地方。
  • class内的自动联想输入： 比如animate类里的 rubberband等如果不是输入在 class 里的时候，不会自动提示，这时候可以写在class里，提示后，复制过来

<view class="clicksbox animated "  hover-class="clickbox-hove rubberBand" hover-stay-time="800"></view>

  • pages.json

    • 首页在pages数组里配置，第一个就是首页

# 顶部导航栏开发

  • 查看官方的pages文档， pages下的style定义 官方的说明是： 用于设置每个页面的状态栏、导航条、标题、窗口背景色等。

  • 接着看到又个 app-plus属性， 官方： 设置编译到 App 平台的特定样式

  • app-plus里又个 titleNView 属性， 是原生导航栏的定义。

  • titleNView里又有 searchInput 和 buttons 定义

  • 按钮里的text实际是图标，来自 iconfront.ttf 这里用utf码的时候，要把 \u 加上


"style": {
	"app-plus":{
		"titleNView":{
			"searchInput":{
				"backgroundColor":"#f5f4f2",
				"align":"center",
				"borderRadius":"2px",
				"placeholder":"搜索文章",
				"placeholderColor":"#6d6c67",
				"disabled": true
				
			},
			"buttons":[
				{
					"text": "搜索",
					"fontSrc":"/static/iconfont.ttf",
					"fontSize":"30px",
					"colorPressed":"#ffffff",
					"text":"\ue614"
				}
			]
		}
	}
}

# Class 与 Style 绑定

  • class支持的语法

<view :class="{ active: isActive }">111</view>
<view class="static" v-bind:class="{ active: isActive, 'text-danger': hasError }">222</view>
<view class="static" :class="[activeClass, errorClass]">333</view>
<view class="static" v-bind:class="[isActive ? activeClass : '', errorClass]">444</view>
<view class="static" v-bind:class="[{ active: isActive }, errorClass]">555</view>
